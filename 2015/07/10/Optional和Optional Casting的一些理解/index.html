Optional， Optional Casting的一些理解

我不知道该翻译什么好，请原谅使用了一些苹果的命名，应该不会造成影响的：）

Optionals可选类型是什么？

在Objective-C中，任意一个类型在有值的时候就是这个类型的值，而没值的时候，默认值就是nil。举个例子：

NSString *abc = [[NSString alloc] init]; //这个时候,abc的值为nil，实际表示的意思是abc的存指针的内存地址里面为0值，不扯远了:)

abc = @”hello,world”; //abc的值为hello，world

这样的情况在Objective-C中太常见了，搞OC的完全不会觉得有什么问题。但是我们这里应该看到一个事实：nil实际上并不是一个字符串，对吧。说了这么多废话就只想说明这一句话。

然后我们再说一个概念，swift是严格类型语言，也就是说，一个类型，就应该只包括这个类型的值。所以hello，world是NSString，nil就不是NSString，他们不是一个层面的东西，都放到NSString里面就不合适了。

理解了概念之后就说到Optional可选类型了。很简单，OC里面把nil和字符串放混到一起是有理由的，因为nil和有值这两种情况是的的确确会存在在程序当中的。而swift严格类型，不能混到一起，怎么办？那就搞个Optional可选类型啦！

Optional可选类型的实质：

enum Optional : NilLiteralConvertible {

case None

case Some(T)

}

如上，Optional可选类型实际上就是一个枚举

是不是有点感觉了，既然一个类里面装不了他们，那就搞个枚举把这几个东西包起来，不就ok了？

可选类型使用？紧跟着数据类型来告诉编译器这个变量可能有值也可能没值,如下:

var age: Int? = nil
var account: Int? = 55

相当于：
var account: Optional = Optional(55)

多问一句，NSString? 和 NSString! 是不是同一个类型？

不是！！！

不同类型的变量可以相互直接赋值么？

不可以！！！

如何使用 ? 和 !

举个例子，我们在OC中UIViewController的一个常用属性：navigationController。相信初学OC的人应该遇到过这样一个报错：

[self.navigationController pushViewController:secondViewController, animated:YES];

这里有一种可能的情况就是忘了初始化UINavigationController来装这个self，所以这个self根本就没有navigationController，也就是self.navigationController = nil。让一个nil执行push方法，当然报错了。

是不是有点感觉了，看看swift里面的做法：
controller.navigationController?.pushViewController(secondViewController, animated: true)

这里的 ? 就表示这navigationController可能是个可选类型，可能有值可能为nil。（这里必须插一条规则：可选类型在执行方法时，如果为nil时，该条语句不执行）

是不是感觉swift这样做的好处了？根据这一条规则，有值，代码正常跑，没值也不会crash！这也就是swift为什么这么整这一套的原因之一了。

而当我们确定这个navigationController在被使用的时候一定有值的，那么写成这样也是可以的：
controller.navigationController!.pushViewController(secondViewController, animated: true)

！紧跟着变量后面，就相当于告诉编译器这个变量是一定有值的，别多操心，就按我说的去跑。

发散想一下，不crash很好，但是如果我想在没有值不执行该方法的时候弄个提示框出来，怎么整？

使用”if let”

先看代码：

if let nav = controller.navigationController {
nav.pushViewController(secondViewController, animated: true)
} else {
//show an alert ot something else
}

使用if let的这一套写法做了次“拆包”，这里说几点：

  1. 这段语句不用解释都能看得懂，所以不解释了

  2. 为什么要这样些，为什么不可以下面那样写？可以，语法上没有问题，但是兄弟，别问那么多了，这是个固定路数，先记下来，以后遇到这种情况就这么整。

    if controller.navigationController == nil {…..}

  3. 这里的nav在这一个管用套路里面，就直接等同于UINavigationController类型了，而不是UINavigationController?类型。

  4. 这里的let也可以用var，没有问题，但基本上都是使用let

使用as? 和 as

这两个东西是主要用于cast（类型转换）。

举例：

FirstViewController继承自UIViewController，并有自己的一个变量topic

class FirstViewController: UIViewController {
var topic: String! = nil
}

在某个方法里，比如在appDelegate里的方法didFinishLaunchingWithOptions里，实现：
let newVC = FirstViewController()

window?.rootViewController = newVC
